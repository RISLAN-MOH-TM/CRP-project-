🚀 INSTALLATION (3 STEPS)
────────────────────────────────────────────────────────────────────────────────

Step 1: Run Installer
──────────────────────────────────────────────────────────────────────────────
Double-click: INSTALL_CVE_RAG.bat

This will:
✅ Check Python installation
✅ Install dependencies (langchain, chromadb, sentence-transformers)
✅ Verify CVE directory
✅ Build vector database
✅ Test search functionality


Step 2: Choose Processing Option
──────────────────────────────────────────────────────────────────────────────
When prompted, choose:

Option 1: All CVEs (100,000+)
   - Time: 2-3 hours
   - Best for: Production use
   - Recommended: Yes

Option 2: Specific year (e.g., 2023)
   - Time: 15-30 minutes
   - Best for: Recent CVEs
   - Recommended: For testing

Option 3: Limited files (e.g., 5000)
   - Time: 5-10 minutes
   - Best for: Quick testing
   - Recommended: First-time setup


Step 3: Start Enhanced Server
──────────────────────────────────────────────────────────────────────────────
cd cve_rag_system
python core\mcp_server_with_cve_rag.py --enable-cve-rag

🐛 TROUBLESHOOTING
────────────────────────────────────────────────────────────────────────────────

Issue: "Python not found"
Solution: Install Python 3.8+ from python.org

Issue: "CVE directory not found"
Solution: Update path in INSTALL_CVE_RAG.bat or provide when prompted

Issue: "Out of memory"
Solution: Choose Option 3 (limited files) or Option 2 (specific year)

Issue: "Slow processing"
Solution: Normal for large datasets. 100,000+ CVEs takes 2-3 hours.
