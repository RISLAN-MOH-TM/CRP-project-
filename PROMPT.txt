PHASE 1 — RECON & SCANNING
───────────────────────────────────────────────────────────────

Run tools in this decision order. Skip only if genuinely not
applicable to what you discover.

  ALWAYS run first (every engagement):
    1. whatweb_scan     — fingerprint tech stack, CMS, frameworks
    2. nmap_scan        — full port scan + service/version + OS (-sCV -T4 -Pn)
    3. nikto_scan       — web server vulnerability scan
    4. gobuster_scan    — directory and file enumeration
    5. nuclei_scan      — CVE template-based detection (critical,high,medium)

  Run if applicable (based on what you find):
    6. ffuf_scan        — if parameters or input fields discovered
    7. sqlmap_scan      — if forms, login pages, or GET/POST params found
    8. hydra_attack     — if a login page is found
    9. wpscan_analyze   — if WordPress detected by whatweb
   10. searchsploit_search — if specific software versions found
   11. subfinder_scan   — if the target is a domain (not bare IP)
   12. metasploit_run   — only if a confirmed exploitable CVE is found

  After EVERY tool output — immediately call CVE RAG tools:
    • Service/version found   → cve_enrich_services("product version, ...")
    • CVE ID in tool output   → cve_lookup("CVE-XXXX-XXXXX")
    • Vuln type confirmed     → cve_search("vuln type product version")

  Do not wait until the end. Enrich in real time as you scan.

  STEP 4 — Render to PDF:
    → render_pdf_report(report_json="[full JSON string from STEP 3]")
