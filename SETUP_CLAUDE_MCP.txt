CLAUDE DESKTOP MCP CONFIGURATION - CVE RAG SYSTEM
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Locate Claude Desktop Config File
──────────────────────────────────────────────────────────────────────────────
Windows: C:\Users\YourUsername\AppData\Roaming\Claude\claude_desktop_config.json

If the file doesn't exist, create it.


STEP 2: Add MCP Server Configuration
──────────────────────────────────────────────────────────────────────────────
Open the config file and add this configuration:

{
  "mcpServers": {
    "kali-pentest-cve-rag": {
      "command": "python",
      "args": [
        "C:\\Users\\User\\User\\Desktop\\mcp\\cve_rag_system\\core\\mcp_server_with_cve_rag.py",
        "--server", "http://YOUR_KALI_IP:5000",
        "--enable-cve-rag",
        "--cve-dir", "C:\\Users\\User\\User\\Desktop\\mcp\\cves"
      ],
      "env": {
        "KALI_API_KEY": "kali-research-project-2026"
      }
    }
  }
}

IMPORTANT: Replace YOUR_KALI_IP with your actual Kali VM IP address!


STEP 3: Restart Claude Desktop
──────────────────────────────────────────────────────────────────────────────
1. Close Claude Desktop completely
2. Reopen Claude Desktop
3. The MCP server will start automatically


STEP 4: Verify Connection
──────────────────────────────────────────────────────────────────────────────
In Claude Desktop, type:

"Check the Kali server health and list available tools"

You should see:
- 18 penetration testing tools (nmap, nikto, sqlmap, etc.)
- 5 CVE RAG tools (search_cve_database, get_cve_details, etc.)

   - Example: "Create enhanced report for 192.168.1.100"

═══════════════════════════════════════════════════════════════════════════════
TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

If MCP server doesn't start:
1. Check Python path is correct
2. Verify Kali server is running
3. Check Kali IP address is correct
4. Ensure .env file has correct API key

If CVE search doesn't work:
1. Verify chroma_cve_db folder exists in cve_rag_system/
2. Check that 319,917 CVEs were indexed
3. Restart Claude Desktop

If connection fails:
1. Ping Kali VM: ping YOUR_KALI_IP
2. Test API: curl http://YOUR_KALI_IP:5000/health
3. Check firewall settings
